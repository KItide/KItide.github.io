注意力机制

发表于 2025-03-17 更新于 2023-03-17 分类于 深度学习

一、什么是注意力

人在观察事物时会有选择性的关注较为重要的信息，称其为注意力。通过持续关注这一关键位置以获得更多的信息，而忽略其他的无用信息，这种视觉注意力机制大大提高了我们处理信息的效率和准确性。深度学习中的注意力机制和人类视觉的注意力机制类似，就是在更多信息中把注意力集中放在重要的点上，选出关键信息，而忽略其他不重要的信息。

上图形象化展示了人类在看到一副图像时如何高效分配有限的注意力资源。其中红色区域表明视觉系统更关注目标，很明显对于上图所示的场景，人们会把注意力更多投入到人的脸部，文本的标题以及文章首句等位置。也

二、注意力机制

注意力机制主要基于查询（Query）、键（Key）和值（Value）这三个核心组件来运作。从本质上讲，它通过计算查询与各个键之间的关联程度，以此确定输入数据中哪些部分与当前任务需求最为相关，进而对相应的值进行加权处理，得到聚焦关键信息后的输出。

2.1 原理

这里我们介绍的是原始的注意力机制，也被称为注意力分数

注意力机制主要基于查询（Query）、**键（Key）和值（Value）**这三个核心组件来运作。从本质上讲，它通过计算查询与各个键之间的关联程度，以此确定输入数据中哪些部分与当前任务需求最为相关，进而对相应的值进行加权处理，得到聚焦关键信息后的输出。

注意力机制该如何实现呢？

​ 首先我（查询对象Q），这张图片（被查询对象V）。我去看这张图片，第一眼，我就回去判断那些东西对我而言更加重要，那些东西对我而言没那么重要,即去计算Q和V里的事务重要程度，也就是相似度计算（谁与谁的依赖关系更强）

这样介绍可能会有一点难以理解，我们可以举一个经典的例子。如下图：

​

​ **Query：**我们就可以将”我”看作成 Query，因为这就是我们当前需要查询的目标，即当前输入的特征表示。

​ Key：可以将每个单词的重要特征表示看作成 Key。

​ **Value：**每个单词本身的特征向量看作为 Value，一般和 Key成对出现，也就是我们常说的”键-值”对。

具体流程如下：

​ 第一阶段：

​ 此阶段主要进行相似度计算。Query（查询向量）分别与多个 Key（键向量，即图中的 Key1 - Key4）通过函数 F (Q,K) 计算相似度 ，得到 s1 - s4 这一系列相似度分数。这里的函数 F 通常是点积运算或者缩放点积运算，其目的是衡量 Query 与每个 Key 之间的相关程度。

​ 第二阶段：

​ 该阶段是类 SoftMax 归一化操作。将阶段 1 得到的相似度分数 s1 - s4 输入到类 SoftMax 函数中，经过 SoftMax 函数处理后，会将这些分数转换为概率分布，即得到 a1 - a4。这些概率值表示了 Query 与每个 Key 的相关程度在整体中的占比，且概率之和为 1。

​ 注：softMax的计算公式为：对于一组实数 (z_1, z_2, …, z_n)，SoftMax函数计算每个元素对应的概率 (p_i) 的公式为：
$$
[p_i=\frac{e^{z_i}}{\sum_{j = 1}^{n}e^{z_j}}]\
(p_i) 是第 (i) 个元素经过SoftMax转换后的概率值。\
(z_i) 是输入的第 (i) 个实数。\
(e) 是自然常数，约等于 (2.71828)。\
(\sum_{j = 1}^{n}e^{z_j}) 表示对所有输入元素取指数后的和。
$$

第三阶段：

​ 这个阶段计算 Attention Value（注意力值）。将阶段 2 得到的概率值 a1 - a4 分别与对应的 Value（值向量，即 Value1 - Value4）进行乘法运算，然后将这些乘积结果相加，最终得到 Attention Value。这个值综合考虑了 Query 与各个 Key 的相关程度，以及对应的 Value 信息 ，是注意力机制的输出结果，体现了对不同 Value 的加权聚合。

3、自注意力机制（self-Attention）

​ 自注意力机制旨在处理序列数据时，打破仅依赖相邻元素的局限，使每个元素都能与序列中的任意元素建立联系。它聚焦于自适应捕捉长程依赖关系，通过量化元素间的相对重要性来实现这一目标。

​ 实际操作中，针对序列里的每一个元素，自注意力机制会先计算它与其余元素的相似度，接着将这些相似度转化为归一化的注意力权重。最后，通过将各元素与对应的注意力权重进行加权求和，生成自注意力机制的输出结果。 其基本架构图如下：

3.1 Embedding操作

3.1.1为什么要进行Embedding呢？

​ 1. 让计算机理解语义 - 计算机不懂文字，Embedding能把像单词这样的东西变成向量。这些向量可以放在一个空间里，语义相近的单词对应的向量距离就近，这样计算机就能理解语义了。比如“猫”和“狗”的向量距离近，因为它们都是动物。

  1. 减少计算量 - 以前表示单词可能用很高维很稀疏的编码，Embedding可以把它变成低维的、密密麻麻的向量。就像把一个很占地方的大东西变成小而紧凑的，计算起来就方便多了。

  2. 方便模型干活和举一反三 - 低维向量更容易被模型处理。就像小零件模型更容易摆弄一样。而且这些向量在大量文字上学到了语义知识，遇到新的文字也能更好地发挥作用。

  3. 结合上下文理解单词 - 一个单词在不同句子里意思可能不同，Embedding可以根据句子上下文来调整单词的向量表示，让它的意思更准确。比如“bank”在讲河的时候和讲钱的时候意思就不一样，Embedding能让它的向量体现出这种差别。

3.2 Q、K操作

​ Embedding操作后， a1,a2,a3,a4 将会作为注意力机制的 input data。

第一步，每个 a1,a2,a3,a4 都会分别乘以三个矩阵，分别是 Q,K,V ；需要注意的是，矩阵Q,K,V在整个过程中是共享的(这也是自注意力自的由来)；公式如下：

​
$$
\begin{align}
q^{i}&=W^{q}a^{i}\
k^{i}&=W^{k}a^{i}\
v^{i}&=W^{v}a^{i}
\end{align}
$$

其中， Q (Query) 的含义一般的解释是用来和其他单词进行匹配，更准确地说是用来计算当前单词或字与其他的单词或字之间的关联或者关系； K(Key) 的含义则是被用来和 Q进行匹配，也可理解为单词或者字的关键信息。

如下图所示，若需要计算 a1 和 a2,a3,a4 之间的关系（或关联），则需要用 q1 和 k2,k3,k4 进行匹配计算，计算公式如下：
$$
α_{ij} = \frac{q^i \cdot k^j}{\sqrt{d_k}}
$$
(当然这个公式并不是固定的，可以用各种方法来计算，大家也能在这方面做创新)

其中， d 表示 Q 和 K 的矩阵维度（在Transformer中一般为512），在 Self-Attention 中， Q和 K的维度是一样的。这里除以 d 的原因是防q 和 k的点乘结果较大，在Transformer中除上d是为使其的方差更加接近于1。

经过 q 和 k 的点乘操作后，会得到 α1,1,α1,2,α1,3,α1,4 ；然后，就是对其进行 softmax 操作，得到 α1,1 ,α1,2 ,α1,3 ,α1,4 。

3.3 V操作

​ $v$ 主要用于表示当前单词或字的重要信息，也能理解成单词的关键特征。比如，$v^1$ 代表 “你” 这个字的重要信息。在涉及 $v$ 的运算里，会把对 $q$、$k$ 操作后得到的 $\tilde{\alpha}{1,1}$，$\tilde{\alpha}{1,2}$，$\tilde{\alpha}{1,3}$，$\tilde{\alpha}{1,4}$ 分别与 $v^1$，$v^2$，$v^3$，$v^4$ 相乘。其计算公式为 $b^1 = \sum_{i} \tilde{\alpha}_{1,i}v^i$ 。

对应的$b^1 = \sum_{i} \tilde{\alpha}_{1,i}v^i$

由此可见，自注意力机制通过计算序列中不同位置之间的相关性（ q,k 操作），为每个位置分配一个权重，然后对序列进行加权求和（ v 操作）。
