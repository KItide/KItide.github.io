DVWA靶场实战

发表于 2023-08-07 更新于 2022-09-09 分类于 教程

dvwa靶场搭建

1.下载配置phpstudy,启动apache和mysql数据库

2.官网下载靶机包，解压修改文件名称为DVWA后放入phpstudy的WWW目录下

3.phpstudy中创建网站 在域名中输入你修改的靶场文件名（DVWA)即可

4.修改配置文件，点开…WWW\DVWA\config目录下的config.inc.php

修改IP（本机127.0.0.1)、库名（默认dvwa不用改）、用户名(默认root)、密码(默认root）、端口

注：user和password来自phpstudy中数据库页面

5.在浏览器中输入localhost/dvwa/即可访问

注：初次访问有登录界面 输入用户名：admin 密码：password即可

渗透流程

信息收集

ip获取

1.nslookup 命令解析域名，获得服务器和地址（判断是否有CDN）

（旁站、c段主机、子域名、端口暂略去）

目录信息收集

dirsearch 扫描以期收获敏感目录文件、源码 等

dirsearch字典如下（可自我收集使用其他字典）

python dirsearch.py –help 获取使用指南，按说明进行命令行书写

python dirsearch.py –url localhost/dvwa/ -e* -i 200,201,201-300 （限定只记录状态码为200-300的目录）

扫描结果在dirsearh 目录下的report中查看

漏洞扫描

1.启动bp点击dashboard-new scan，选择扫描类型（crwl and audit,爬取并审计）

选择目标网址（靶场网址 ）注意！不可扫描其他正常网址，否则属于非法渗透，有风险。

选择协议类型（默认http,可自定义）

2.点击第二个扫描设置选项

点击New-crwaling

设置任务名称（dvwa crl),将下面选项依次展开确认（可不更改）后点击save

点击 new - audit,同理，设置名称，展开确认后save

点击ok，开始扫描

如图，可在dashboard板右侧issue activity框或任务进程右下角的viewdetails中查看漏洞详细情况

选中全部条目后右键选择report selected issues生成扫描报告

一直next到选择文件

点开保存路径，查看漏洞详细信息

登录页面简单密码爆破

获取请求

用burpsuite内置浏览器（用其他浏览器需设置bp代理）打开dvwa，设置漏洞等级low后进入brute force页面

开启burpsuite拦截功能，如图

返回浏览器，乱输一个密码后点击login,bp拦截到请求后自动弹出页面

右键送至intruder模块

设置攻击参数

1.设置攻击密码内容为攻击字段（默认已知用户名admin）

首先点击clear$，关闭intruder模块的绿色分节符，然后设置password为分节符（点击Add)

2..选择攻击模式sniper

Sniper :狙击手 精准打击，适合单参数

3.设置pyloads

选择payloadtype为simplelist (对应sniper)

在payloadoptions中选择bp自带字典（点击add from list),选择合适字典

点击starttack开始爆破

第一条 长度明显不同

双击，在response中发现 “Welcome to the password protected area admin”，确认爆破成功

csrf- token获取

将dvwa靶场漏洞等级调整为 high后查看登录页面源代码（view resource)

发现此处相对于低等级进行了安全加固，设置了anti-csrf策略

Anti-CSRF token:每一次访问，都需要一个新的Token,一次使用，用完销毁，防止第三方网站冒充身份进行攻击。

获取这一次的响应来进行 下一次访问

依次轮流的访问

1.流程同上，打开bp代理,乱输入密码，点击登录，获取请求，发送intruder,为密码和token字段设置分节符

设置attack type为pitchfork（草叉 多个payload和攻击字段）

payload1（password）为simple list

payload2(token字段)为recrusvive （递归查找，拿到token)

intruder-options里找到grep-extract,点击add,点击feth获取一个响应，在响应包中找到token字段双击选中

上方自动生成匹配规则

复制该字段（不复制就会被销毁）,后点击ok，将字段粘贴到initial…中

安全加固

token策略

作为计算机术语时，token是“令牌”的意思。Token是服务端生成的一串字符串，以作客户端进行请求的一个令牌，当第一次登录后，服务器生成一个Token便将此Token返回给客户端，以后客户端只需带上这个Token前来请求数据即可，无需再次带上用户名和密码.

相对于完全裸露的密码登录方式，token策略在一定程度上抵御了csrf(跨站请求伪造）攻击，避免第三方网站利用被攻击网站生效的cookie，直接对服务器接口发起请求。

同时，设置token复杂的生成规则可延缓两次请求的间隔时间，从而大大提升密码爆破的难度，降低了其效率。
